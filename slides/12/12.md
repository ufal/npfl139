# MuZero

The MuZero algorithm extends the AlphaZero by a **trained model**, alleviating
the requirement for a known MDP dynamics. It is evaluated both on board games
and on the Atari domain.

~~~
At each time-step $t$, for each of $0 вүӨ k вүӨ K$ steps, a model $Ој_Оё$ with
parameters $Оё$, conditioned on past observations $o_1, вҖҰ, o_t$ and future
actions $a_{t+1}, вҖҰ, a_{t+k}$, predicts three future quantities:
~~~
- the policy $вҶ’p^k_t вҲј ПҖ(a_{t+k+1} | o_1, вҖҰ, o_t, a_{t+1}, вҖҰ, a_{t+k})$,
~~~
- the value function $v^k_t вҲј рқ”ј\big[u_{t+k+1} + Оі u_{t+k+2} + вҖҰ | o_1, вҖҰ, o_t, a_{t+1}, вҖҰ, a_{t+k}\big]$,
~~~
- the immediate reward $r^k_t вҲј рқ”ј\big[u_{t+k} | o_1, вҖҰ, o_t, a_{t+1}, вҖҰ, a_{t+k}\big]$,

where $u_i$ are the observed rewards and $ПҖ$ is the behaviour policy.

~~~
- Note that we use the notation from the paper, with actions indexed by $t+1$
  instead of $t$ compared to our usual notation.

# MuZero

At each time-step $t$ (omitted from now on for simplicity), the model is composed of three components:
a **representation** function, a **dynamics** function, and a **prediction** function.

~~~
- The dynamics function, $(r^k, s^k) вҶҗ g_Оё(s^{k-1}, a^k)$, simulates the MDP
  dynamics and predicts an immediate reward $r^k$ and an internal state $s^k$.
  The internal state has no explicit semantics, its only goal is to accurately
  predict rewards, values, and policies.

~~~
- The prediction function, $(вҶ’p^k, v^k) вҶҗ f_Оё(s^k)$, computes the policy and the
  value function, similarly as in AlphaZero.

~~~
- The representation function, $s^0 вҶҗ h_Оё(o_1, вҖҰ, o_t)$, generates an internal
  state encoding from the past observations.

# MuZero Model Architecture

For board games,
- the prediction function, $(вҶ’p^k, v^k) вҶҗ f_Оё(s^k)$, consists of two heads, each
  comprising a convolutional layer, a fully connected layer in the value head,
  and a convolutional/linear output layer (as in AlphaZero);
~~~
- both the representation function, $s^0 вҶҗ h_Оё(o_1, вҖҰ, o_t)$, and the dynamics
  function, $(r^k, s^k) вҶҗ g_Оё(s^{k-1}, a^k)$, consist of an initial convolution
  followed by 16 residual blocks (analogous to AlphaZero, but with 16 instead of 19
  blocks).

~~~
For Go and shogi, the last 8 board positions are passed to the representation
function; for chess, 100 board positions are used to allow correct prediction of
draws.

~~~
To encode an action for the dynamics function, one-hot representation is used
in Go;
~~~
for chess, an action is represented by a binary indicator of the position
a piece was moved from, then two target position indicators (the position
a piece was moved to, if on the board, and binary indication of whether the
target was valid), and 5 planes indicating the type of promotion (fourВ piece types
or none);
~~~
shogi is analogous to chess.

---
# MuZero Model Architecture

For the Atari domain,
- the prediction function also consists of a convolutional layer, followed by
  a linear output layer to the size of the output;
~~~
- the representation function is a sequence of convolutions reducing the
  observation resolution from $96Г—96$ to $48Г—48$ using a single convolution
  with stride 2 and 128 filters, followed by 2 residual blocks with 128 filters,
  a convolution with 256 filters and stride 2 with output resolution $24Г—24$,
  3 residual blocks with 256 filters, average pooling with stride 2 with
  output resolution $12Г—12$, 3 residual blocks with 256 filters, with a final
  average pooling with stride 2 producing the final $6Г—6$ resolution;
~~~
- the dynamics function is a sequence of residual blocks operating on the $6Г—6$
  resolution.

~~~
The inputs to the representation function are last 32 observations (3 channels
each) and the actions that led to them (each represented as a channel with value $a/18$);
actions are encoded as one-hot planes for the dynamics function.

# MuZero вҖ“ MCTS

The MCTS algorithm is very similar to the one used in AlphaZero, only the
trained model is used. It produces a policy $вҶ’ПҖ_t$ and a value estimate $ОҪ_t$.

~~~
- All actions, including the invalid ones, are allowed at any time, except at
  the root, where the invalid actions (available from the current state) are
  disallowed.

~~~
- No states are considered terminal during the search.

~~~
- During the backup phase, we consider a general discounted bootstrapped return
  $$G_k = вҲ‘\nolimits_{t=0}^{l-k-1} Оі^t r_{k+1+t} + Оі^{l-k} v_l.$$

~~~
- Furthermore, the expected return is generally unbounded. Therefore, MuZero
  normalizes the Q-value estimates to $[0, 1]$ range by using the minimum and
  the maximum of the values present in the search tree:
  $$QМ„(s, a) = \frac{Q(s, a) - \min_{s',a' вҲҲ \mathrm{Tree}} Q(s', a')}{\max_{s',a' вҲҲ \mathrm{Tree}} Q(s', a') - \min_{s',a' вҲҲ \mathrm{Tree}} Q(s', a')}.$$

---
# MuZero вҖ“ Action Selection

To select a move, we employ the MCTS algorithm and then sample
an action from the obtained policy, $a_{t+1} вҲј вҶ’ПҖ_t$.

~~~
For games, the same strategy of sampling the actions $a_t$ as in AlphaZero is used.
~~~
In the Atari domain, the actions are sampled according to the visit counts for the
whole episode, but with a given temperature $T$:
$$ПҖ(a|s) = \frac{N(s, a)^{1/T}}{вҲ‘_b N(s, b)^{1/T}},$$
where $T$ is decayed during training вҖ“ for first 500k steps it is 1, for the
next 250k steps it is 0.5 and for the last 250k steps it is 0.25.

~~~
While for the board games 800 simulations are used during MCTS, only 50 are used
for Atari.

~~~
In case of Atari, the replay buffer consists of 125k sequences of 200 actions.

# MuZero вҖ“ Training

During training, we utilize a sequence of $K$ moves. We estimate the return
using bootstrapping as $z_t = u_{t+1} + Оі u_{t+2} + вҖҰ + Оі^{n-1} u_{t+n} + Оі^n ОҪ_{t+n}$.
The values $K=5$ and $n=10$ are used in the paper, with batch size 2048 for the
board games and 1024 for Atari.

~~~
The loss is then composed of the following components:
$$рқ“ӣ_t(Оё) = вҲ‘_{k=0}^K рқ“ӣ^r (u_{t+k}, r_t^k) + рқ“ӣ^v(z_{t+k}, v^k_t) + рқ“ӣ^p(вҶ’ПҖ_{t+k}, вҶ’p^k_t) + c \|\theta\|^2.$$

~~~
In Atari, rewards are transformed using $\sign(x)\big(\sqrt{|x| + 1} - 1\big) + Оөx$ for $Оө=10^{-3}$,
and the authors utilize a cross-entropy loss with 601 categories for values $-300, вҖҰ, 300$, which they claim
to be more stable (similar to how C51 distributional RL works; but we do not
utilize predicted distribution, only its mean).

~~~
Furthermore, in Atari the discount factor $Оі=0.997$ is used, and the replay buffer elements
are sampled according to prioritized replay with priority $вҲқ |ОҪ-z|^Оұ$; importance sampling
with exponent $ОІ$ is used to account for changing the sampling distribution
($Оұ=ОІ=1$ is used).

# AlphaZero as Regularized Policy Optimization

Recall that in AlphaZero, actions are selected according to a variant of PUCT
algorithm:
$$a^* = \argmax\nolimits_a \bigg(Q(s, a) + C(s) P(s, a) \frac{\sqrt{N(s)}}{1 + N(s, a)}\bigg),$$
with a slightly time-increasing exploration rate
$C(s) = \log\left(\frac{1+N(s)+19625}{19625}\right) + 1.25 вүҲ 1.25$.

~~~
The paper _Jean-Bastien Grill et al.: Monte-Carlo Tree Search as Regularized
Policy Optimization_, the authors have shown how to interpret this algorithm
as a regularized policy optimization.


---
# AlphaZero as Regularized Policy Optimization

Policy optimization is usually an iterative procedure, which in every step
improves a current policy $ПҖ_{вҶ’Оё_0}$ according to
$$ПҖ_{вҶ’Оё'} вүқ \argmax_{вҶ’y вҲҲ рқ“ў} вҶ’q_{ПҖ_{вҶ’Оё_0}}^\T вҶ’y - рқ“Ў(вҶ’y, ПҖ_{вҶ’Оё_0}),$$
where $рқ“ў$ is a $|рқ“җ|$-dimensional simplex and $рқ“Ў \colon рқ“ў^2 вҶ’ в„қ$ is an optional
(usually convex) regularization term.

~~~
- with $рқ“Ў = 0$, the above reduces to policy iteration (used for example in DQN);
~~~
- with $рқ“Ў = 0$, if the policy is updated using a single gradient step, the
  algorithm reduces to policy gradient;
~~~
- when $рқ“Ў(вҶ’y, ПҖ_{вҶ’Оё_0}) = -H(вҶ’y)$, we recover the Soft Actor Critic objective;
~~~
- for $рқ“Ў(вҶ’y, ПҖ_{вҶ’Оё_0}) = D_\textrm{KL}(ПҖ_{вҶ’Оё_0} \| вҶ’y)$ we get an analogue of
  the TRPO objective, which motivated PPO;
~~~
- the MPO algorithm (which we did not discuss) employs $рқ“Ў(вҶ’y, ПҖ_{вҶ’Оё_0}) = D_\textrm{KL}(вҶ’y \| ПҖ_{вҶ’Оё_0})$.

---
# AlphaZero as Regularized Policy Optimization

Let us define the **empirical visit distribution** $ПҖМӮ$ as
$$ПҖМӮ(a|s) вүқ \frac{1 + N(s, a)}{|рқ“җ| + вҲ‘_b N(s, b)}.$$

~~~
The added plus ones makes the following analysis easier, but are not strictly
necessary.

~~~
We also define the **multiplier** $О»_N$ as
$$О»_N(s) вүқ C(s) вӢ… \frac{\sqrt{вҲ‘_b N(s, b)}}{|рқ“җ| + вҲ‘_b N(s, b)}.$$

~~~
With these definitions, we can rewrite the AlphaZero action selection to
$$a^* = \argmax\nolimits_a \bigg(Q(s, a) + О»_N вӢ… \frac{ПҖ_{вҶ’Оё}(a|s)}{ПҖМӮ(a|s)}\bigg).$$

---
# AlphaZero as Regularized Policy Optimization

$$a^* = \argmax\nolimits_a \bigg(Q(s, a) + О»_N вӢ… \frac{ПҖ_{вҶ’Оё}(a|s)}{ПҖМӮ(a|s)}\bigg)$$

For notational simplicity, we will represent $Q(s, a)$ as a vector $вҶ’q$, where
$q_a = Q(s, a)$, and similarly the policies as $вҶ’ПҖ_{вҶ’Оё}$, $вҶ’ПҖМӮ$.

~~~
Furthermore, for two vectors $вҶ’a, вҶ’b$, let $\frac{вҶ’a}{вҶ’b}$ denote element-wise
division with $(\frac{вҶ’a}{вҶ’b})_i вүқ \frac{a_i}{b_i}.$ 

~~~
With this notation, the action selection can be succinctly written as
$$a^* = \argmax\nolimits_a \Big(вҶ’q + О»_N \frac{вҶ’ПҖ_{вҶ’Оё}}{вҶ’ПҖМӮ}\Big).$$

---
# AlphaZero as Regularized Policy Optimization

Let $вҶ’ПҖМ„$ be the solution of the following objective:
$$вҶ’ПҖМ„ вүқ \argmax\nolimits_{вҶ’y вҲҲ рқ“ў} \Big(вҶ’q^\T вҶ’y - О»_N D_\textrm{KL}(вҶ’ПҖ_{вҶ’Оё} \| вҶ’y)\Big).$$

~~~
The solution to this objective can be computed explicitly as
$$вҶ’ПҖМ„ = О»_N \frac{вҶ’ПҖ_{вҶ’Оё}}{Оұ - вҶ’q},$$
where $Оұ вҲҲ в„қ$ is set (using binary search) such that the result is a proper distribution.
~~~
- Note that $Оұ вүҘ \max_{bвҲҲрқ“җ} \big(q_b + О»_N ПҖ_{вҶ’Оё}(b)\big)$, because $ПҖМ„(a)$ must
  be at most 1.
~~~
- Furthermore, $Оұ вүӨ \max_{bвҲҲрқ“җ} (q_b) + О»_N$, because we need $вҲ‘_a ПҖМ„(a) = 1$
  and we combine $вҲ‘_a \frac{О»_N ПҖ_{вҶ’Оё}(a)}{\max_b (q_b) + О»_N - q_a} вүӨ вҲ‘_a \frac{О»_N ПҖ_{вҶ’Оё}(a)}{О»_N} = 1$
  with the fact that $вҲ‘_a \frac{О»_N ПҖ_{вҶ’Оё}(a)}{Оұ - q_a}$ is a decreasing
  function of $Оұ вүҘ \max_b q_b$.

~~~
Note the $О»_N вүҲ 1/\sqrt N$ decreasing the regularization for increasing number
of simulations.

---
# AlphaZero as Regularized Policy Optimization

In the paper, it is proven that the action $a^*$ selected by the AlphaZero algorithm fulfills
$$a^* = \argmax\nolimits_a \bigg(\frac{вҲӮ}{вҲӮ N(s, a)} \Big(вҶ’q^\T вҶ’ПҖМӮ - О»_N D_\textrm{KL}(вҶ’ПҖ_{вҶ’Оё} \| вҶ’ПҖМӮ)\Big)\bigg).$$

~~~
In other words, $ПҖМӮ$ вҖңtracksвҖқ $ПҖМ„$.

~~~
Furthermore, it can be also shown that for the selected action $a^*$,
$$ПҖМӮ(a^*|s) вүӨ ПҖМ„(a^* | s),$$
until in the limit, the two distributions coincide.

~~~
If you are interested in the proof, see Appendix D (pages 19-22).

# AlphaZero as Regularized Policy Optimization

The $ПҖМ„$ can be used in the AlphaZero algorithm in several ways:

- **Act**: the action in self-play games could be sampled according to
  $ПҖМ„(вӢ…|s_\textrm{root})$ instead of $ПҖМӮ$;

~~~
- **Search**: during search, we could sample the actions stochastically

~~~
- **Learn**: we could use $ПҖМ„$ as the target policy during training instead

~~~
- **All**: all of the above.

# Gumbel-Max Trick

Let $z$ be a categorical variable with class probabilities $вҶ’p = (p_1, p_2, вҖҰ, p_K)$.

~~~
The Gumbel-Max trick (based on a 1954 theorem from E. J. Gumbel) states that
we can draw samples $z вҲј вҶ’p$ using
$$z = \operatorname{one-hot}\bigg(\argmax_{iвҲҲ\{1, вҖҰ, K\}} \big(g_i + \log p_i\big)\bigg),$$

![w=38%,f=right](gumbel_vs_normal.svgz)

where $g_i$ are independent samples drawn from the $\operatorname{Gumbel}(0, 1)$
distribution.

~~~
To sample $g$ from the distribution $\operatorname{Gumbel}(0, 1)$, we can sample
$u вҲј U(0, 1)$ and then compute
$$g = -\log(-\log u).$$

---

# Gumbel Distribution

First recall that exponential distribution $\operatorname{Exp}(О»)$ has
$$\operatorname{PDF}(x; О») = О» e^{-О»x},~~~\operatorname{CDF}(x; О») = 1 - e^{-О»x}.$$

~~~
The standard $\operatorname{Gumbel}(0, 1)$ distribution has
$$\operatorname{PDF}(x) = e^{-x - e^{-x}},~~~\operatorname{CDF}(x) = e^{-e^{-x}}.$$

~~~
The Gumbel distribution can be used to model the distribution of maximum
of a number of samples from the exponential distribution:
~~~
if $xМғ$ is a maximum of $N$ samples from the $\operatorname{Exp}(1)$
distribution, we get that
$$P(xМғ - \log N вүӨ x)
  = P(xМғ вүӨ x + \log N)
  = \operatorname{CDF}_{\operatorname{Exp}(1)}\big(x + \log N\big)^N
  = \Big(1 - \frac{e^{-x}}{N}\Big)^N,$$
~~~
which converges to $e^{-e^{-x}} = \operatorname{CDF}_{\operatorname{Gumbel}(0, 1)}(x)$ for $N вҶ’ вҲһ$.

---

# Gumbel-Max Trick Proof

To prove the Gumbel-Max trick, we first reformulate it slightly.

Let $l_i$ be logits of a categorical distribution (so that the class probabilities
$ПҖ_i вҲқ e^{l_i}$), and let $g_i вҲј \operatorname{Gumbel}(0, 1)$. Then
$$ПҖ_k = P\big(k = \argmax\nolimits_i (g_i + l_i)\big).$$

~~~
We first observe that the theorem is invariant to a scalar shift of logits,
so we can without loss of generality assume that $вҲ‘_i e^{l_i} = 1$ and $ПҖ_i
= e^{l_i}$.

~~~
For convenience, denote $u_i вүқ g_i + l_i$.

~~~
We will use both the $\operatorname{PDF}$ and $\operatorname{CDF}$ of
a $\operatorname{Gumbel}(0, 1)$ distribution:
$$\begin{aligned}
  \operatorname{PDF}(g_i) &= e^{-g_i - e^{-g_i}}, \\
  \operatorname{CDF}(g_i) &= e^{-e^{-g_i}}.
\end{aligned}$$

---

# Gumbel-Max Trick Proof

To finish the proof, we compute

$\displaystyle\kern12em{}\mathllap{P\big(k = \argmax\nolimits_i (g_i + l_i)\big)} = P(u_k вүҘ u_i, вҲҖ_{iвү k})$

~~~
$\displaystyle\kern12em{} = вҲ« P(u_k) вҲҸ\nolimits_{iвү k} P(u_k вүҘ u_i | u_k) \d u_k$

~~~
$\displaystyle\kern12em{} = вҲ« P(g_k | g_k = u_k - l_k) вҲҸ\nolimits_{iвү k} P(g_i вүӨ u_k - l_i | u_k) \d u_k$

~~~
$\displaystyle\kern12em{} = вҲ« e^{\textcolor{blue}{l_k}-u_k-e^{\textcolor{darkgreen}{l_k}-u_k}} вҲҸ\nolimits_{iвү k} e^{-e^{\textcolor{magenta}{l_i}-u_k}} \d u_k$

~~~
$\displaystyle\kern12em{} = вҲ« \textcolor{blue}{ПҖ_k}e^{-u_k-\textcolor{darkgreen}{ПҖ_k}\textcolor{red}{e^{-u_k}}} вҲҸ\nolimits_{iвү k} e^{-\textcolor{magenta}{ПҖ_i}\textcolor{red}{e^{-u_k}}} \d u_k$

~~~
$\displaystyle\kern12em{} = ПҖ_k вҲ« e^{-u_k-\textcolor{red}{e^{-u_k}} вҲ‘_i ПҖ_i} \d u_k$

~~~
$\displaystyle\kern12em{} = ПҖ_k вҲ« e^{-g_k-e^{-g_k}} \d g_k = ПҖ_k вӢ… 1 = ПҖ_k.$

# Gumbel AlphaZero and MuZero

Let $ПҖ$ be a categorical distributions parametrized with $\mathit{logits}(a)$.
Let $вҶ’g вҲҲ в„қ^k$ be a vector of independent Gumbel(0, 1) random variables.

~~~
The Gumbel-Max trick states that
$$A = \argmax\nolimits_a \big(g(a) + \mathit{logits}(a)\big)$$
has a distribution $A вҲј ПҖ$.

~~~
The Gumbel-Max trick can be generalized to **Gumbel-Top-k** trick, capable of
producing $n$ actions without replacement by considering the top $n$ scoring
actions $\operatorname{argtop}(вҶ’g + \boldsymbol{logits}, n)$:
$$\begin{aligned}
  A_1 &= \argmax\nolimits_a \big(g(a) + \mathit{logits}(a)\big), \\
  A_2 &= \argmax\nolimits_{a вү  A_1} \big(g(a) + \mathit{logits}(a)\big), \\
  \vdots \\
  A_n &= \argmax\nolimits_{a \notвҲҲ \{A_1, вҖҰ, A_{n-1}\}} \big(g(a) + \mathit{logits}(a)\big). \\
\end{aligned}$$

---
# GumbelZero, A. Guaranteed Policy Improvement

## A. Guaranteed Policy Improvement

For a small number of simulations, PUCT does not guarantee policy improvement.
~~~
- Consider for example three actions with prior policy $(50\%, 30\%, 20\%)$ and
action values $(0, 0, 1)$.
~~~
- The PUCT rule will select the first two actions.
~~~
- However, the value function of any policy considering just the first two actions
  is 0, which is worse than the value function of the prior policy.

~~~
In GumbelZero, we start by sampling $n$ actions without replacement using the
Gumbel-Max trick with Gumbel noise $вҶ’g$.

~~~
Our first attempt is to define a one-hot policy selecting an action $A_{n+1}$
such that
$$A_{n+1} = \argmax_{a вҲҲ \{A_1, вҖҰ, A_n\}} \big(g(a) + \mathit{logits}(a) + Пғ(q(a))\big),$$
where $Пғ$ can be any monotonically increasing transformation.

# GumbelZero, A. Guaranteed Policy Improvement

The policy choosing the action
$$A_{n+1} = \argmax_{a вҲҲ \{A_1, вҖҰ, A_n\}} \Big(g(a) + \mathit{logits}(a) + Пғ\big(q(a)\big)\Big)$$
guarantees policy improvement, i.e., $рқ”ј[q(A_{n+1})] вүҘ рқ”ј_{aвҲјПҖ}[q(a)]$.

~~~
Considering a fixed Gumbel noise $вҶ’g$, it follows that
$$q\bigg(\argmax_{a вҲҲ \{A_1, вҖҰ, A_n\}} \Big(g(a) + \mathit{logits}(a) + Пғ\big(q(a)\big)\Big)\bigg) вүҘ q\bigg(\argmax_{a вҲҲ \{A_1, вҖҰ, A_n\}} \Big(g(a) + \mathit{logits}(a)\Big)\bigg),$$
~~~
- either the action chosen on both sides is the same and we get an equality, or
~~~
- the action on the left side is different, meaning it has larger $q(a)$.

~~~
Finally, if the inequality holds for any $вҶ’g$, it holds also in expectation.
With the Gumbel-Max trick transforming the expectation of the right side to sampling
an action $a вҲј ПҖ$, we get the required $рқ”ј[q(A_{n+1})] вүҘ рқ”ј_{aвҲјПҖ}[q(a)]$.

# GumbelZero, B. Planning on Stochastic Bandit

## B. Planning on Stochastic Bandit

When we get only an estimate $qМӮ(a)$ of the action-value function, it is probably
beneficial to visit an action multiple times.

~~~
Furthermore, choosing actions in the root using a UCB-like rule is not optimal:
~~~
- UCB minimizes cumulative regret, i.e., it maximizes the sum of the obtained
  returns;

~~~
- in the root our goal is to obtain the best possible $A_{n+1}$, i.e.,
  to maximize just $рқ”ј[q(A_{n+1})]$.

~~~
The authors evaluated several simple regret minimization algorithms, and they chose
Sequential Halving (because it was easier to tune and does not have
problem-dependent parameters).

# GumbelZero, C. Better Improved Policy

## C. Better Improved Policy
Using a one-hot policy based on
$$A_{n+1} = \argmax_{a вҲҲ \{A_1, вҖҰ, A_n\}} \big(g(a) + \mathit{logits}(a) + Пғ(q(a))\big)$$
results in using a simple policy loss
$$L_\textrm{simple}(ПҖ) = -\log ПҖ(A_{n+1}).$$

---
style: .katex-display { margin: .9em 0 }
# GumbelZero, C. Better Improved Policy

However, more information from the search might be extracted by using all action-value functions
$q(a)$ produced by the search.

~~~
- First, we complete the action values using
  $$\mathit{completedQ}(a) вүқ \begin{cases} q(a) & \textrm{if~}N(a) > 0, \\ v_ПҖ
  & \textrm{otherwise}.\end{cases}$$

~~~
- Then, we define improved policy as
  $$ПҖ' = \softmax\big(\mathit{logits}(a) + Пғ(\mathit{completedQ}(a))\big).$$

~~~
  It can be again proven (appendix C of the paper) that $ПҖ'$ is an improved policy, so $рқ”ј_{aвҲјПҖ'}[q(a)] вүҘ рқ”ј_{aвҲјПҖ}[q(a)]$.

~~~
- A natural loss is then
  $$L_\textrm{completed}(ПҖ) = D_\textrm{KL}(ПҖ' \| ПҖ).$$

---
# GumbelZero, C. Better Improved Policy

The authors propose to use
$$Пғ\big(qМӮ(a)\big) вүқ \big(c_\textrm{visit} + \max_b N(b)\big) c_\textrm{scale} qМӮ(a),$$
for $c_\textrm{visit} = 50$, $c_\textrm{scale} = 1.0$.

~~~
Furthermore, the authors propose a consistent approximation to $v_ПҖ$ based on
a network-predicted $vМӮ_ПҖ$ and the $q(a)$ of the visited actions:
$$v_\textrm{mix} вүқ \frac{1}{1 + вҲ‘_b N(b)} \Bigg(vМӮ_ПҖ + \Big(вҲ‘\nolimits_b N(b)\Big)\frac{вҲ‘_{a, N(a) > 0} ПҖ(a) q(a)}{вҲ‘_{a, N(a) > 0} ПҖ(a)}\Bigg).$$

~~~
Overall, the algorithm denoted in the paper as Gumbel MuZero utilizes Sequential
Halving with Gumbel and trains using the improved policy combining logits
and action values completed by $v_\textrm{mix}$; otherwise it is the same as
MuZero.

# GumbelZero, D. Action Selection in Non-Root Nodes

## D. Action Selection in Non-Root Nodes

We might consider utilizing the improved policy $ПҖ'$ also in the non-root nodes,
by for example sampling actions from it. Additionally, the authors provide
a deterministic algorithm of choosing non-root actions minimizing the
difference between $ПҖ'$ and the current visit counts:
$$a^* = \argmin_a вҲ‘_b \Bigg(ПҖ'(b) - \underbrace{\frac{N(b) + [a = b]}{1 + вҲ‘_c N(c)}}_{\textrm{normalized visit counts if taking~}a}\Bigg)^2.$$

~~~
This formula can be simplified to
$$a^* = \argmax_a \Bigg(ПҖ'(a) - \frac{N(a)}{1 + вҲ‘_b N(b)}\Bigg).$$

~~~
When this action selection is used, the authors call the algorithm
Full Gumbel MuZero.
